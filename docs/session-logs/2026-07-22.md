## 2. Part A — Console & cell changes

### 2.1 Table height → 1.0 m arm-base-to-table gap
- Measured: table sits **1000 mm** below the arm-base origin (entered 1010, corrected
  to 1000 = 100 cm).
- `config/local_config.yaml` `workspace` override: `dims_mm[2]` and
  `mount.base_xyz_mm[2]` → `1000`; invariant `base_z == dims_mm[2]` kept; X/Y footprint
  (2000×750) unchanged. Gap `H = base_z − table_z = 1000 mm` flows to `robot_bridge` +
  the viewer via `libs.qc_config.load_config()` / `GET /api/config`.
- `MOUNT_H` is import-time → needs a **server restart**; the viewer reads it at
  page-load → needs a **page reload**.

### 2.2 Reach reality check — "the part is too low" (no code)
Base 1.0 m up, SR5 reach 919 mm ⇒ a part flat on the table is at/beyond the practical
scanning envelope (sweet-spot ≈ 300–700 mm above the table). **Parts must sit on a
raised fixture/pedestal.** Left as an open design decision (feeds §2 placeholder in
Part B).

### 2.3 Part centred in the sim
`scripts/export_viewer_bundle.py`: the grounding shift now also **centres the footprint
in X,Y** under the base (was Z-only; identity `corner_transform` left it ~(−167,+192) mm
off). One shared shift moves mesh + waypoints + dome + box together. Viewer-only.

### 2.4 Home pose in joint space + return-home on every scan
- **`HOME_JOINTS_DEG = [-71.38, 93.33, 127.11, 157.33, -61.23, -77.06]`** (env override
  `QC_HOME_JOINTS_DEG`) — TCP (−88.1, 21.4, −553.5) mm / RPY (87.2, −0.7, 1.2) deg in
  `endInRef`, read live from the arm.
- Stored as **joints** on purpose: homing is **MoveJ** (`move_joints`, gated by
  `QC_ALLOW_MOTION` only). Cartesian MoveL is disabled on the real arm and singular here.
- New `go_home()` = MoveJ + `_wait_settle`. Wired into both scan entrypoints
  (`start_scan_trace`, `start_follow_path`): **home first** (mandatory), **home at the
  end** on normal completion, **skipped after operator abort/E-stop**.

### 2.5 Home button fixed
Was hardcoded to `move {joints:[0,0,0,0,0,0]}` — never used the home pose. New route
**`POST /api/robot/home`** → `move_joints(HOME_JOINTS_DEG)` (non-blocking). Front-end
`home()` posts it; single source of truth shared with the scan homing.

### 2.6 Hardcoded J2 rail guard
`move_joints` **refuses** any J2 target outside **[−120°, 115°]** (`J2_MIN_DEG`/`J2_MAX_DEG`
in `robot_bridge.py`); UI `JOINTS0` J2 max 120→115 to match. Guards **joint-space**
commands (jog/home) only — a Cartesian move lets the controller choose J2, which is
exactly why Part B (own IK) exists.

### 2.7 Jog-to-targets fixed, then two-column
- **Bug:** the command field shared `j.v` with live telemetry, held only by a 4-second
  global timer; if it lapsed before Jog, a poll overwrote the typed target → Jog sent
  the *current* pose → no motion.
- **Fix:** per-joint `edited` hold (target persists until Jog clears it). Then split the
  row into a read-only **live POSITION** column and an editable **COMMAND target**
  column, so telemetry and target never fight.

## 3. Part B — Own the IK (ROS 2 + MoveIt)

**Why:** IK lived in the ROKAE controller, so J2 could route into the mounting frame
*mid-path* and we'd only find out by watching. Moving IK into our stack (MoveIt) makes
every arm command a **planned joint trajectory** that is collision-checked against the
gantry **before** execution, gives an **accurate sim** (same joints), and is the target
architecture (`docs/architecture.md`: PathPlanner owns MoveIt, MovementDriver streams
joints). Everything below runs in the `qc-humble` Docker container (host has no ROS 2).

### 3.1 Model + collision (Phase 1) — PROVEN
- `qc_cell.urdf.xacro`: mount 1.2→**1.0 m** + the **gantry as fixed collision boxes** —
  plate 1.65×0.41×0.01 m at the mount plane, two 0.08×0.04 m posts at X=±0.785 (from
  Ra's measured cell: 80×40 extrusion, 1650 outside-to-outside, 10 mm plate, centred).
- `qc_cell.srdf` (new): vendor arm self-collision matrix + gantry exceptions — disable
  plate↔base, plate↔link1, plate↔posts (the arm is bolted 1 cm under the plate); **keep
  plate↔link2-6 enabled** = the J2-into-rail collision we must catch.
- `qc_move_group.launch.py`: loads the `qc_cell` URDF **and** SRDF; IK = **KDL**.
- **Proof** (`test/gantry_collision_proof.py` vs live move_group): zero pose
  `valid=True` (no false plate↔base hit); `J2=1.1 rad → valid=False`, contact
  `xMateSR5_link4 ↔ gantry_post_px`.

### 3.2 Plan + execute + orchestrate (Phases 2/4/5) — PROVEN on the mock
- `qc_mission.launch.py` brings up rosbridge + move_group + arm_driver + path_planner +
  movement_driver + task_manager (+ scanner/inspection).
- `/mission/plan` → TaskManager → PathPlanner runs CAD→coverage→frame→MoveIt and
  publishes a ScanPath + `/plan/trajectory`; `/execute_path` → MovementDriver plays it
  on the mock ArmDriver and the arm reaches the last point.

### 3.3 Console preview (Phase 3) — code done, browser-unverified
`gui/viewer/ros_bridge.js` (new, raw-WebSocket rosbridge client, no roslib dep);
`viewer3d.js playJointTrajectory()` poses the arm directly from MoveIt's planned joints
(accurate sim, no per-frame IK); `viewer.js` subscribes `/plan/trajectory` +
`/mission/state`. Additive + silently retries. **Needs a browser to confirm** (no JS
runtime on the dev box). Also: the **gantry is now rendered** in `viewer3d.js` so the
operator sees what J2 can hit.

### 3.4 Collision-checked jog — mechanism PROVEN
The console pre-flights each jog target against MoveIt `/check_state_validity` over
rosbridge and refuses self/gantry collisions (`ros_bridge.js` gained `callService`;
console `_checkCollision` + `jogToTargets` guard; fail-open + warn if the ROS graph is
down). **Proven headless over rosbridge** (a stdlib WebSocket client mimicking the JS):
`J2=1.1 rad → invalid` (contact link4↔gantry_post), zero pose → valid. In-browser JS
wiring still needs a browser to confirm.

## 4. Problems encountered & how they were resolved

| # | Problem (symptom) | Root cause | Fix | Verified |
|---|---|---|---|---|
| 1 | Home button drove the arm to all-zeros, not the home pose | Front-end hardcoded `joints:[0,0,0,0,0,0]`; never referenced `HOME_JOINTS_DEG` | New `/api/robot/home` route → `move_joints(HOME_JOINTS_DEG)`; button posts it | mock: zeros → stored home |
| 2 | Jog-to-targets did nothing | Typed target (`j.v`) clobbered by live-telemetry poll after a 4 s timer lapsed | Per-joint `edited` hold + split live/target columns | code review; brace-balanced |
| 3 | Arm self-collided when **jogging** | Jog bypasses MoveIt entirely — no collision checking on manual joint commands | Collision-checked jog via `/check_state_validity` over rosbridge | headless rosbridge: colliding target refused |
| 4 | PathPlanner crashed: "signal only works in main thread" | `gmsh.initialize()` installs a SIGINT handler off the action-callback thread | `gmsh.initialize(interruptible=False)` in `cad_loader` | e2e: coverage ran, ScanPath published |
| 5 | MovementDriver couldn't move the mock: "move rejected: motors off" | Driver never powered the arm before playback | `_ensure_powered()` (`/arm/set_power` True) before executing | e2e: 5-pt trajectory reached last point |
| 6 | MoveIt config not committable | `.gitignore` `config/` (unanchored) swallowed `ros2_ws/.../qc_moveit_config/config/` | Anchor to `/config/` (root only); MoveIt config now tracked | `git check-ignore`; files committed |
| 7 | Scan trajectory empty: **cartesian fraction 0.00** | Part at raw CAD origin — over the singular column & mostly below reach (IK 5/12); cartesian started from the arm's straight-down pose | (a) `_reachable_shift` places footprint at `scan_anchor_m` + raises to `pedestal_z_m`; (b) IK-seed each line's cartesian `start_state` | **IK 5/12→12/12; fraction 0.00→1.00, 1,196 pts** |
| 8 | Requested 5 cm buffer broke the scan (fraction 1.00→0.14) | The arm is bolted ~1 cm under the plate, so a 5 cm robot pad self-collides + folds fail | Reverted default pad to 0 (true geometry is already collision-free); collision-checked jog is the real safety win | e2e: padding 0 → fraction 1.00 |
| 9 | TracIK unavailable | `ros-humble-trac-ik-*` not in the Humble apt repo | Use MoveIt's built-in KDL (achieves fraction 1.00); TracIK from source deferred | KDL plan verified |
| 10 | gmsh needed in-container (no gmsh, no pip) | Base image has neither; slow apt mirror | Copy host's `gmsh.py` + `libgmsh.so.4.15` (pure ctypes) onto `PYTHONPATH` | `import gmsh; initialize()` OK in-container |
