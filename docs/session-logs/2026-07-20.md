## What changed, in order

### 1. Second scan primitive — "Rectangle" (table-aligned box)
- New `libs/path_planning/box_raster.py` and `scripts/plan_box.py`.
- `fit_table_box()` fits the smallest box that sits **flat on the table** (vertical
  axis = table normal; only the footprint **yaw** is optimised via a min-area
  rectangle / rotating calipers). Not a free min-volume OBB — Ra wanted it flat.
- `generate_box_waypoints()` rasters each face at standoff, aimed square-on inward;
  **skips the down-facing (table) face** (unscannable, and lifting its waypoints was
  pushing them up into tall parts).
- Emits a `box` record (centre / half_dims / **quaternion**); `export_viewer_bundle.py`
  transforms it (oriented `BoxGeometry`) and sets a `primitive` field on the bundle.
- Viewport toggle **Hemisphere ⇄ Rectangle** re-plans on the selected primitive via
  new `POST /api/plan/primitive`. Backend planner map: `dome`→hemisphere, `box`→plan_box;
  `--target-waypoints` withheld from both shell planners; `--orient-deg` passed to both.

### 2. Table clearance (kept the head off the table)
- Iterated: delete-below-15 cm → lift-to-15 cm → back to delete → finally **two
  mechanisms**:
  - Generic `libs/path_planning/table_clearance.py` (called in `scanpath_convert.py`):
    DELETES waypoints within a clearance. **Default 0 = OFF** (`QC_TABLE_CLEARANCE_MM`).
  - Rectangle-specific **lift** in `plan_box.py` (`--min-clearance-mm`, default 150,
    `QC_BOX_CLEARANCE_MM`): raises low waypoints to 15 cm, re-aimed at the part.
- Planners now record `table_up_mm` (placed-frame table plane): `plan_hemisphere`,
  `plan_box`, `plan_surround`, `plan_lawn`.

### 3. "Waypoints inside the model" bug — fixed
- Root cause was NOT the planner (every waypoint is provably standoff-outside the
  enclosing shape). It was the **Debug flip buttons rotating the mesh without
  re-planning** — the mesh swung into the path laid out for the old pose. Made Debug
  flips **re-plan** (like the main viewer). Also fixed a real bug where lifting the
  box's bottom-face waypoints pushed them into tall parts (now that face is skipped).

### 4. Bold orange arm-travel line (Debug)
- `debug.js drawArmPath()`: a `TubeGeometry` (bold, since WebGL ignores line width)
  through the waypoints **in visit order**, so it shows the arm's real traversal
  including the jumps between rings/faces.

### 5. Part-number CAD database
- Backend catalogue now = **`/home/frank/Desktop/Parts`** (618 files, env
  `QC_PARTS_DIR`) instead of `config/cad`.
- Console STEP 1: **search/autocomplete by part number** (substring over the DB,
  ≤30 matches, error on no match); the old file upload kept as a secondary option.

### 6. "Confirm & start scan" now traces the viewport path
- Console `sendToRobot` switched from `/api/robot/follow_path` (position-only, wasn't
  moving the arm) to **`/api/robot/scan_trace`** — the same oriented trace the Debug
  viewport uses (which works). `scan_trace` now falls back to `LAST_PLAN.orient_deg`
  for its IK soft-bias when the request omits it.
- New **"Cancel scan & clear path"** button → `POST /api/robot/stop` (aborts the
  trace, clears the queued path, returns to Confirm).

### 7. Viewport tidy-up
- **Generate path** button added to both viewports.
- **Layers** toggles MOVED from the main viewport into Debug.
- **Arm comms monitor** removed from the main viewport (Debug-only).
- Drag-to-place ("move the part on the table") was built earlier in the session then
  **removed** at Ra's request — including its backend `/api/plan/place` + offset plumbing.
